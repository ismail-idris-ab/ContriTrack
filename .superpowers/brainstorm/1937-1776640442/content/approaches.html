A — Status Banner + Smart Form

Show all active circles at the top with paid/unpaid badges. The upload form only appears for the currently selected unpaid circle. If a circle is already paid, the selector shows a green receipt view instead of the form.

Pros

  • Least code change from current
  • Clear per-circle status at a glance
  • No red error messages

Cons

  • Form still collapses/shows based on selection — can feel jumpy

B — One Tab Per Circle

Each active circle is its own tab. Paid tabs show a receipt/confirmation view. Unpaid tabs show the full upload form. User can freely switch between circles and see each one's status independently.

Pros

  • Crystal clear — each circle is isolated
  • Easy to scan all circle statuses
  • No confusion about which circle the form is for

Cons

  • More UI work to build
  • Tabs can get cramped with many circles

C — Two-Panel (Sidebar + Form)

Left panel lists all active circles with their status. Selecting a paid circle shows a receipt view on the right. Selecting an unpaid circle shows the upload form on the right. Status is the primary UI, upload is secondary.

Pros

  • Most professional, cleanest mental model
  • Status is front-and-center, not an error message
  • Scales well to many circles

Cons

  • Most UI work to build
  • Two-panel layout may feel complex on mobile

Click a card to select. My recommendation: Option A for quick fix, Option B for best UX.
